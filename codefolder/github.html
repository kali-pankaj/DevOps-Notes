GitHub Zero to Hero

1. Basics of GitHub Commands

  • git init: Initializes a new Git repository in current directory.
  • git clone [url]: Clones an existing repository from a URL to the local machine.
  • git status: Shows the status of changes in the working directory status (which file is modified staged etc.)
  • git add [file]: Stages changes for the next commit.
  • git commit -m "message": Commits staged changes with a message.
  • git push: Pushes commits to a remote repository.
  • git pull: Fetches and merges changes from a remote repository.
  • git fetch: Fetches changes from a remote repository without merging them.
  • git merge [branch]: Merges changes from a specific branch into the current branch.
  • git log: Shows the commit history of the repository.
  • git diff: Shows the differences between staged and unstaged changes.

2. Branching Commands (Managing Branches)

  • git branch: Lists all local branches in the repository.
  • git branch [branch-name]: Creates a new branch with the given name.
  • git checkout [branch-name]: Switches to the specified branch.
  • git checkout -b [branch-name]: Creates and switches to a new branch with the given name.
  • git merge [branch-name]: Merges changes from the specified branch into the current branch.
  • git rebase [branch-name]: Reapplies commits from the current branch on top of the specified branch.
  • git branch -d [branch-name]: Deletes the specified local branch.
  • git branch -D [branch-name]: Force deletes the specified local branch.
  • git remote set-head origin [branch-name]: Sets the default branch for a remote repository.
  • git delete [branch-name]: Deletes the specified local branch.

3. Remote Commands (Managing Remote and Syncing with the cloud)

  • git remote -v: Lists all configured remote repositories and their URLs.
  • git remote add [name] [url]: Adds a new remote repository with the specified name and URL.
  • git remote remove [name]: Removes the specified remote repository.
  • git push [remote] [branch]: Pushes the specified branch to the specified remote repository.
  • git push -u [remote] [branch]: Pushes the specified branch to the specified remote repository and sets the upstream tracking.
  • git pull [remote] [branch]: Fetches and merges changes from the specified branch of the specified remote repository.
  • git fetch [remote]: Fetches changes from the specified remote repository without merging them.
  • git remote show [remote-name]: Shows information about the specified remote repository.

4. Staging and Committing (managing changes in the working directory)

  • git add [file]: Stages changes for the next commit.
  • git add -A: Stages all changes (modified, deleted, and new files) for the next commit.
  • git reset [file]: Unstages changes for the next commit.
  • git commit --amend: Amends the last commit with staged changes.
  • git commit --all: Commits all changes (modified, deleted, and new files) with a default message.
  • git commit -a: Commits all changes (modified, deleted, and new files) with a default message.
  • git commit --no-verify: Commits staged changes without running pre-commit hooks.
  • git reset --soft HEAD~1: Resets the last commit but keeps changes staged.
  • git reset --hard HEAD~1: Resets the last commit and discards all changes.
  • git commit -m "message": Commits staged changes with a message.
  • git status: Shows the status of files in the working directory and staging area.
  • git diff [file]: Shows differences between staged and unstaged changes in a specific file.

5. Viewing and Comparing Changes (inspecting changes)

  • git diff: Shows differences between the working directory and the staging area.
  • git diff --staged: Shows differences between the staging area and the last commit.
  • git log: Shows the commit history.
  • git log --oneline: Shows the commit history in a compact format.
  • git log --graph: Shows the commit history as a graph.
  • git show [commit]: Shows details of a specific commit.
  • git diff HEAD~1 HEAD: Shows differences between the last two commits.
  • git log --author="name": Shows commits by a specific author.
  • git log --since="date": Shows commits since a specific date.
  • git blame [file]: Shows the author of each line in a file.

6. Reverting and Resetting Changes

  • git reset --soft HEAD~1: Resets the last commit but keeps changes staged.
  • git reset --hard HEAD~1: Resets the last commit and discards all changes.
  • git reset [commit]: Resets the current branch to a specific commit.
  • git revert [commit]: Creates a new commit that undoes changes from a previous commit.
  • git clean -f: Removes untracked files from the working directory.
  • git clean -fd: Removes untracked files and directories from the working directory.
  • git restore --staged [file]: Unstages a specific file from the staging area.
  • git reset --mixed HEAD~1: Resets the last commit but keeps changes in the working directory.
  • git reset --merge HEAD~1: Resets the last commit and tries to preserve changes in the working directory.

7. Stashing Changes (Temporary saving of changes)

  • git stash: Stashes all changes in the working directory.
  • git stash save "message": Stashes changes with a custom message.
  • git stash list: Lists all stashed changes.
  • git stash show: Shows the details of the most recent stash.
  • git stash apply: Applies the most recent stash to the working directory.
  • git stash pop: Applies the most recent stash and removes it from the list.
  • git stash drop: Removes the most recent stash from the list.
  • git stash clear: Removes all stashed changes.
  • git stash drop [stash]: Removes a specific stash from the list.
  • git stash branch [branchname] [stash]: Creates a new branch and applies a specific stash to it.

8. Tag (Managing versions of your code)

  • git tag: Lists all tags in the repository.
  • git tag [tag-name]: Creates a new tag with the specified name.
  • git tag -d [tag-name]: Deletes the specified tag.
  • git tag -a [tag-name] -m "message": Creates an annotated tag with a message.
  • git push [remote] [tag-name]: Pushes a specific tag to a remote repository.
  • git push --tags: Pushes all tags to a remote repository.
  • git show [tag-name]: Shows details of a specific tag.
  • git checkout [tag-name]: Checks out a specific tag (detached HEAD state).
  • git tag -l "pattern": Lists tags that match a specific pattern.
  • git fetch --tags: Fetches all tags from the remote repository.

9. Git Config and Info (configuration and repository information)

  • git config --global user.name "name": Sets the global username for Git commits.
  • git config --global user.email "email": Sets the global email for Git commits.
  • git config --list: Lists all Git configuration settings.
  • git config --global core.editor [editor]: Sets the default text editor for Git commands.
  • git config --global merge.tool [tool]: Sets the default merge tool for Git.
  • git config --global diff.tool [tool]: Sets the default diff tool for Git.
  • git config --global alias.[alias-name] [command]: Creates a custom alias for a Git command.
  • git rev-parse HEAD: Shows the commit hash of the current HEAD.
  • git rev-parse --abbrev-ref HEAD: Shows the name of the current branch.
  • git config --global color.ui auto: Enables colored output in Git commands.
  • git config --global alias.st status: Creates an alias for `git status`.

10. Git Aliases (Custom Commands)

  • git config --global alias.[alias-name] [command]: Creates a custom alias for a Git command.
  • git st: Alias for `git status` (if configured).
  • git co: Alias for `git checkout` (if configured).
  • git br: Alias for `git branch` (if configured).
  • git ci: Alias for `git commit` (if configured).
  • git lg: Alias for a custom log format (if configured).
  • git amend: Alias for `git commit --amend` (if configured).
  • git unstage: Alias for `git reset [file]` (if configured).
  • git last: Alias for `git log -1 HEAD` (if configured).
  • git graph: Alias for a custom log graph format (if configured).
  • git clean -n: Shows what files would be removed by `git clean` without actually removing them.
  • git clean -f: Removes untracked files from the working directory.
  • git clean -fd: Removes untracked files and directories from the working directory.
  • git gc: Cleans up unnecessary files and optimizes the local repository.

11. Git Hooks (Automated scripts triggered by Git actions)

  • git init: Initializes a new Git repository.
  • git pre-commit: Runs before a commit is created.
  • git post-commit: Runs after a commit is created.
  • git post-merge: Runs after a merge is completed.

12. Git submodule (Managing external repositories within a a project)

  • git submodule add [repository-url] [path]: Adds a new submodule to the repository.
  • git submodule update --init: Initializes and updates all submodules.
  • git submodule update --remote: Updates all submodules to the latest commit from their respective remote repositories.
  • git submodule status: Shows the current status of all submodules.
  • git submodule foreach [command]: Runs a command in each submodule.
  • git submodule deinit [path]: Deinitializes a specific submodule.
  • git rm --cached [path]: Removes a submodule from the index (but not from the working directory).
  • git commit -m "message": Commits changes related to submodules with a message.
  • git push: Pushes changes related to submodules to the remote repository.
  • git submodule init: Initializes submodules in the working directory.
  • git submodule update: Updates submodules to the latest commit in their respective repositories.

13. Git Workflow commands (working with other and managing collaboration)

  • git pull: Fetches and merges changes from a remote repository.
  • git push: Pushes commits to a remote repository.
  • git fetch: Fetches changes from a remote repository without merging them.
  • git merge [branch]: Merges changes from a specific branch into the current branch.
  • git rebase [branch]: Reapplies commits from the current branch on top of the specified branch.
  • git cherry-pick [commit]: Applies changes from a specific commit to the current branch.
  • git revert [commit]: Creates a new commit that undoes changes from a previous commit.
  • git stash: Stashes all changes in the working directory to save them temporarily.
  • git stash apply: Applies stashed changes back to the working directory.
  • git stash pop: Applies stashed changes and removes them from the stash list.
  • git pull --rebase: Fetches and rebases changes from a remote repository.
  • git rebase --abort: Aborts an ongoing rebase process.
  • git rebase --continue: Continues an ongoing rebase process after resolving conflicts.
  • git reset [commit]: Resets the current branch to a specific commit.
  • git merge --no-ff [branch]: Merges changes from a specific branch without creating a fast-forward commit.
  • git pull --no-commit: Fetches and merges changes from a remote repository without committing them.
  • git push --force-with-lease: Forces pushing changes to a remote repository, but only if the remote branch hasn't been updated by others.
  • git push --force: Forces pushing changes to a remote repository, overriding any existing changes.
  • git cherry-pick [commit]: Applies changes from a specific commit to the current branch.

14> Git Merge Strategies(Handling merge conflicts and strategies)

  • git merge [branch]: Merges changes from a specific branch into the current branch.
  • git merge --no-ff [branch]: Merges changes from a specific branch without creating a fast-forward commit.
  • git merge --squash [branch]: Merges changes from a specific branch but squashes all commits into a single commit.
  • git merge --strategy=recursive [branch]: Uses the recursive merge strategy (default).
  • git merge --strategy=ours [branch]: Uses the "ours" strategy, which keeps the current branch's changes in case of conflicts.
  • git merge --strategy=theirs [branch]: Uses the "theirs" strategy, which keeps the incoming branch's changes in case of conflicts.
  • git merge --abort: Aborts an ongoing merge process and resets the state to before the merge started.
  • git merge --continue: Continues an ongoing merge process after resolving conflicts.
  • git mergetool: Launches a visual tool to help resolve merge conflicts.
  • git diff --name-only --diff-filter=U: Lists files with merge conflicts.
  • git status: Shows the status of files in the working directory, including any merge conflicts.
  • git add [file]: Stages resolved files after fixing merge conflicts.

15. Git bisect (Finding the commit that introduced a bug)

  • git bisect start: Starts the bisect process.
  • git bisect bad [commit]: Marks a commit as bad (introduced the bug).
  • git bisect good [commit]: Marks a commit as good (before the bug was introduced).
  • git bisect reset: Resets the bisect process to the original state.
  • git bisect log: Shows the log of bisect process.
  • git bisect visualize: Visualizes the current state of the bisect process.

16. Git hooks for Automation (pre-commit and post commit hooks)

  • pre-commit hooks: Scripts that run before a commit is made. They can be used to enforce code quality standards, run tests, or prevent commits with issues.
  • post-commit hooks: Scripts that run after a commit is made. They can be used to perform actions such as sending notifications, updating documentation, or triggering continuous integration builds.
  • pre-push hooks: Scripts that run before changes are pushed to a remote repository. They can be used to run tests, check for code quality, or prevent pushing changes that don't meet certain criteria.
  • post-merge hooks: Scripts that run after a merge is completed. They can be used to perform actions such as updating dependencies, running tests, or sending notifications about the merge.

17. Git clean and prune (removing untracked files and stale remote-tracking branches)

  • git clean -f: Removes untracked files from the working directory.
  • git clean -fd: Removes untracked files and directories from the working directory.
  • git clean -n: Shows what files would be removed by `git clean` without actually removing them.
  • git prune: Removes stale remote-tracking branches that no longer exist on the remote repository.
  • git prune --dry-run: Shows which branches would be pruned without actually pruning them.
  • git prune --expire [time]: Prunes branches that are older than the specified time.
  • git prune --aggressive: Prunes branches more aggressively, potentially removing more branches than the default behavior.
  • git gc: Garbage collects unused objects and optimizes the repository.

18. Git Archive (Creating a snapshot of Repository Contents)

  • git archive [branch-name] --format=zip -o [filename].zip: Creates a zip archive of the specified branch.
  • git archive [branch-name] --format=tar -o [filename].tar: Creates a tar archive of the specified branch.
  • git archive --format=tar [branch-name] | gzip > [filename].tar.gz: Creates a gzipped tar archive of the specified branch.
  • git archive --format=zip HEAD -o latest.zip: Creates a zip archive of the current HEAD.
  • git archive --format=tar HEAD | gzip > latest.tar.gz: Creates a gzipped tar archive of the current HEAD.
  • git archive --format=zip --prefix=[folder-name]/ HEAD -o latest.zip: Creates a zip archive with a specific folder prefix.
  • git archive --format=tar --prefix=[folder-name]/ HEAD | gzip > latest.tar.gz: Creates a gzipped tar archive with a specific folder prefix.
  • git archive --format=zip [commit] -o [filename].zip: Creates a zip archive of a specific commit.
  • git archive --format=tar [commit] | gzip > [filename].tar.gz

19. Git blame and annotate (tracing the history )

  • git blame [file]: Shows the author of each line in a file along with the commit hash and timestamp.
  • git annotate [file]: Alias for `git blame` (if configured).
  • git blame -L [start],[end] [file]: Shows blame information for a specific range of lines in a file.
  • git blame -w [file]: Ignores whitespace changes when blaming a file.
  • git blame --reverse [file]: Shows blame information in reverse order (from the most recent commit to the oldest).
  • git blame --show-name [file]: Shows the filename along with the blame information.
  • git blame --show-email [file]: Shows the email address of the author along with the blame information.
  • git blame --show-number [file]: Shows the line number along with the blame information.
  • git blame --show-date [file]: Shows the date of each commit along with the blame information.
  • git blame --show-summary [file]: Shows a summary of each commit along with the blame information.
  • git blame --show-stats [file]: Shows statistics about the commits that modified each line in a file.

20. Git reflog (tracking changes in the head)

  • git reflog: Shows the history of changes to the HEAD reference.
  • git reflog show [branch-name]: Shows the reflog for a specific branch.
  • git reflog show --date=short [branch-name]: Shows the reflog for a specific branch with short dates.
  • git reflog show --all: Shows the reflog for all branches.
  • git reflog delete [ref]: Deletes a specific reference from the reflog.
  • git reflog expire --expire=now: Expires all reflog entries that are older than the specified time.
