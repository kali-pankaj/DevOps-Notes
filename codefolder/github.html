GitHub Zero to Hero

1. Basics of GitHub Commands
  • git init: Initializes a new Git repository in current directory.
  • git clone [url]: Clones an existing repository from a URL to the local machine.
  • git status: Shows the status of changes in the working directory.
  • git add [file]: Stages changes for the next commit.
  • git commit -m "message": Commits staged changes with a message.
  • git push: Pushes commits to a remote repository.
  • git pull: Fetches and merges changes from a remote repository.
  • git fetch: Fetches changes without merging.
  • git merge [branch]: Merges a branch into the current branch.
  • git log: Shows commit history.
  • git diff: Shows differences between changes.
2. Branching Commands (Managing Branches)
  • git branch: Lists all local branches.
  • git branch [branch-name]: Creates a new branch.
  • git checkout [branch-name]: Switches branch.
  • git checkout -b [branch-name]: Create & switch branch.
  • git merge [branch-name]: Merge branch.
  • git rebase [branch-name]: Reapply commits.
  • git branch -d [branch-name]: Delete branch.
  • git branch -D [branch-name]: Force delete branch.
  • git remote set-head origin [branch-name]: Set default remote branch.
  • git delete [branch-name]: Delete branch.
3. Remote Commands (Managing Remote and Syncing)
  • git remote -v: Lists remotes.
  • git remote add [name] [url]: Adds remote.
  • git remote remove [name]: Removes remote.
  • git push [remote] [branch]: Push branch.
  • git push -u [remote] [branch]: Push & set upstream.
  • git pull [remote] [branch]: Pull branch.
  • git fetch [remote]: Fetch remote.
  • git remote show [remote-name]: Show remote info.
4. Staging and Committing (Managing Changes in Working Directory)
  • git add [file]: Stages changes for the next commit.
  • git add -A: Stages all changes.
  • git reset [file]: Unstages changes.
  • git commit --amend: Amends the last commit.
  • git commit --all: Commits all changes with default message.
  • git commit -a: Commits all modified files.
  • git commit --no-verify: Commit without running hooks.
  • git reset --soft HEAD~1: Reset last commit, keep staged.
  • git reset --hard HEAD~1: Reset last commit, discard changes.
  • git commit -m "message": Commit with message.
  • git status: Show file status.
  • git diff [file]: Show file differences.
5. Viewing and Comparing Changes (Inspecting Changes)
  • git diff: Compare working directory and staging.
  • git diff --staged: Compare staging and last commit.
  • git log: Show commit history.
  • git log --oneline: Compact history view.
  • git log --graph: Graph view of commits.
  • git show [commit]: Show commit details.
  • git diff HEAD~1 HEAD: Compare last two commits.
  • git log --author="name": Filter by author.
  • git log --since="date": Filter by date.
  • git blame [file]: Show line author info.
6. Reverting and Resetting Changes
  • git reset --soft HEAD~1: Reset commit, keep staged.
  • git reset --hard HEAD~1: Reset commit and delete changes.
  • git reset [commit]: Reset to specific commit.
  • git revert [commit]: Undo commit safely.
  • git clean -f: Remove untracked files.
  • git clean -fd: Remove untracked files and directories.
  • git restore --staged [file]: Unstage specific file.
  • git reset --mixed HEAD~1: Reset commit, keep working files.
  • git reset --merge HEAD~1: Reset but preserve working state.
7. Stashing Changes (Temporary Saving)
  • git stash: Stash all changes.
  • git stash save "message": Stash with message.
  • git stash list: List stashes.
  • git stash show: Show stash details.
  • git stash apply: Apply stash.
  • git stash pop: Apply and remove stash.
  • git stash drop: Delete recent stash.
  • git stash clear: Delete all stashes.
  • git stash drop [stash]: Delete specific stash.
  • git stash branch [branchname] [stash]: Create branch from stash.
8. Tag (Managing Versions)
  • git tag: List all tags.
  • git tag [tag-name]: Create tag.
  • git tag -d [tag-name]: Delete tag.
  • git tag -a [tag-name] -m "message": Annotated tag.
  • git push [remote] [tag-name]: Push specific tag.
  • git push --tags: Push all tags.
  • git show [tag-name]: Show tag info.
  • git checkout [tag-name]: Checkout tag (detached HEAD).
  • git tag -l "pattern": List tags by pattern.
  • git fetch --tags: Fetch remote tags.
9. Git Config and Info (Configuration & Repository Info)
  • git config --global user.name "name": Set username.
  • git config --global user.email "email": Set email.
  • git config --list: List config settings.
  • git config --global core.editor [editor]: Set default editor.
  • git config --global merge.tool [tool]: Set merge tool.
  • git config --global diff.tool [tool]: Set diff tool.
  • git config --global alias.[alias-name] [command]: Create alias.
  • git rev-parse HEAD: Show current commit hash.
  • git rev-parse --abbrev-ref HEAD: Show branch name.
  • git config --global color.ui auto: Enable color output.
  • git config --global alias.st status: Alias for status.
10. Git Aliases (Custom Commands)
  • git config --global alias.[alias-name] [command]: Create custom alias.
  • git st: Alias for git status.
  • git co: Alias for git checkout.
  • git br: Alias for git branch.
  • git ci: Alias for git commit.
  • git lg: Alias for custom log.
  • git amend: Alias for commit amend.
  • git unstage: Alias for reset file.
  • git last: Alias for last commit.
  • git graph: Alias for log graph.
  • git clean -n: Preview clean.
  • git clean -f: Remove untracked files.
  • git clean -fd: Remove files and directories.
  • git gc: Optimize repository.
11. Git Hooks (Automated Scripts Triggered by Git Actions)
  • git init: Initializes a new Git repository.
  • pre-commit: Runs before a commit is created.
  • post-commit: Runs after a commit is created.
  • post-merge: Runs after a merge is completed.
12. Git Submodule (Managing External Repositories)
  • git submodule add [repository-url] [path]: Adds a submodule.
  • git submodule update --init: Initialize and update submodules.
  • git submodule update --remote: Update to latest remote commit.
  • git submodule status: Show submodule status.
  • git submodule foreach [command]: Run command in each submodule.
  • git submodule deinit [path]: Deinitialize submodule.
  • git rm --cached [path]: Remove submodule from index.
  • git commit -m "message": Commit submodule changes.
  • git push: Push submodule changes.
  • git submodule init: Initialize submodules.
  • git submodule update: Update submodules.
13. Git Workflow Commands (Collaboration & Team Work)
  • git pull: Fetch and merge remote changes.
  • git push: Push commits.
  • git fetch: Fetch without merging.
  • git merge [branch]: Merge branch.
  • git rebase [branch]: Reapply commits.
  • git cherry-pick [commit]: Apply specific commit.
  • git revert [commit]: Undo commit safely.
  • git stash: Temporarily save changes.
  • git stash apply: Apply stash.
  • git stash pop: Apply and remove stash.
  • git pull --rebase: Pull with rebase.
  • git rebase --abort: Cancel rebase.
  • git rebase --continue: Continue rebase.
  • git reset [commit]: Reset to commit.
  • git merge --no-ff [branch]: Non fast-forward merge.
  • git pull --no-commit: Pull without auto commit.
  • git push --force-with-lease: Safe force push.
  • git push --force: Force push.
  • git cherry-pick [commit]: Apply specific commit.
14. Git Merge Strategies (Handling Merge Conflicts)
  • git merge [branch]: Merge branch.
  • git merge --no-ff [branch]: Non fast-forward merge.
  • git merge --squash [branch]: Squash merge.
  • git merge --strategy=recursive [branch]: Default recursive strategy.
  • git merge --strategy=ours [branch]: Keep current branch changes.
  • git merge --strategy=theirs [branch]: Keep incoming branch changes.
  • git merge --abort: Abort merge.
  • git merge --continue: Continue merge.
  • git mergetool: Launch visual merge tool.
  • git diff --name-only --diff-filter=U: List conflict files.
  • git status: Show conflict status.
  • git add [file]: Stage resolved files.
15. Git Bisect (Finding Buggy Commit)
  • git bisect start: Start bisect.
  • git bisect bad [commit]: Mark bad commit.
  • git bisect good [commit]: Mark good commit.
  • git bisect reset: Reset bisect process.
  • git bisect log: Show bisect log.
  • git bisect visualize: Visualize bisect.
16. Git Hooks for Automation
  • pre-commit hooks: Run before commit.
  • post-commit hooks: Run after commit.
  • pre-push hooks: Run before push.
  • post-merge hooks: Run after merge.
17. Git Clean and Prune
  • git clean -f: Remove untracked files.
  • git clean -fd: Remove files and directories.
  • git clean -n: Preview clean.
  • git prune: Remove stale branches.
  • git prune --dry-run: Preview prune.
  • git prune --expire [time]: Expire old refs.
  • git prune --aggressive: Aggressive prune.
  • git gc: Garbage collection.
18. Git Archive (Repository Snapshot)
  • git archive [branch] --format=zip -o file.zip: Create zip archive.
  • git archive [branch] --format=tar -o file.tar: Create tar archive.
  • git archive --format=tar [branch] | gzip > file.tar.gz: Create gzipped tar.
  • git archive --format=zip HEAD -o latest.zip: Archive current HEAD.
  • git archive --format=tar HEAD | gzip > latest.tar.gz: Archive HEAD tar.gz.
  • git archive --format=zip --prefix=folder/ HEAD -o latest.zip: Archive with prefix.
  • git archive --format=tar --prefix=folder/ HEAD | gzip > latest.tar.gz: Tar with prefix.
  • git archive --format=zip [commit] -o file.zip: Archive specific commit.
19. Git Blame and Annotate
  • git blame [file]: Show line author info.
  • git annotate [file]: Alias for blame.
  • git blame -L start,end [file]: Blame line range.
  • git blame -w [file]: Ignore whitespace.
  • git blame --reverse [file]: Reverse order blame.
  • git blame --show-name [file]: Show filename.
  • git blame --show-email [file]: Show email.
  • git blame --show-number [file]: Show line number.
  • git blame --show-date [file]: Show date.
  • git blame --show-summary [file]: Show summary.
  • git blame --show-stats [file]: Show stats.
20. Git Reflog (Tracking HEAD Changes)
  • git reflog: Show HEAD history.
  • git reflog show [branch]: Show branch reflog.
  • git reflog show --date=short [branch]: Reflog with dates.
  • git reflog show --all: Show all reflogs.
  • git reflog delete [ref]: Delete reflog entry.
  • git reflog expire --expire=now: Expire old entries.

Git For DevOps

1. Git for Continuous Integration / Continuous Deployment (CI/CD)
  • git fetch --all: Fetch all remote branches.
  • git merge origin/main: Merge latest main branch changes.
  • git push --force-with-lease: Safe force push.
  • git rebase origin/main: Reapply commits on latest base.
  • git tag [version_name]: Create release tag.
2. Git for Infrastructure as Code (IaC)
  • git clone [iac_repo_url]: Clone IaC repository.
  • git checkout [branch_name]: Switch branch.
  • git pull origin [branch_name]: Update infrastructure code.
  • git submodule update --init: Initialize submodules.
3. Git Hooks for DevOps Automation
  • pre-commit hooks: Run before commit.
  • post-commit hooks: Run after commit.
  • pre-push hooks: Run before push.
  • post-merge hooks: Run after merge.
4. Git for Feature Flag Management
  • git branch feature/[feature-name]: Create feature branch.
  • git checkout feature/[feature-name]: Switch branch.
  • git merge feature/[feature-name]: Merge feature.
  • git push origin feature/[feature-name]: Push for review.
  • git tag [feature-name]-release: Tag feature release.
5. Git for Multi-Environment Deployment
  • git checkout [environment_branch]: Switch environment.
  • git merge --no-ff [environment_branch]: Controlled merge.
  • git push origin [environment_branch]: Push environment branch.
  • git tag [environment]-[version]: Tag environment version.
6. Git and Deployment Automation
  • git push: Trigger deployment pipeline.
  • git pull: Update deployed application.
  • git tag [version]: Tag release version.
  • git merge [branch]: Merge to trigger deployment.
  • git reset --hard [commit_hash]: Rollback deployment.
7. GitOps Workflow

GitOps uses Git as the single source of truth for infrastructure and application configuration.

  • git clone [repo_url]: Clone GitOps repository.
  • git checkout [branch]: Switch environment branch.
  • git pull: Sync latest configuration.
  • git commit -m "message": Commit changes.
  • git push origin [branch]: Trigger automated deployment.
  • git tag [version]: Tag for version control & rollback.
8. Advantages of Git for CI/CD Pipelines
  • git push --tags: Manage version releases.
  • git fetch --tags: Sync release tags.
  • git log --oneline --graph: Visualize history.
  • git diff [commit1] [commit2]: Compare versions.
  • Pull Requests: Enable code review & collaboration.
9. Git and Docker Integration

Git stores Dockerfiles and configuration. CI/CD builds Docker images from Git commits.

  • git clone [repo_url]: Clone Docker project.
  • git checkout [branch]: Switch build branch.
  • git tag [version]: Trigger image version build.
  • git merge [branch]: Merge to trigger build pipeline.
  • git reset --hard [commit_hash]: Rollback Docker deployment.
  • git push origin [branch]: Trigger image build.
10. Git and Secrets Management

Integrates with tools like Vault or cloud secret managers to avoid exposing sensitive data.

  • git rm --cached [file]: Remove sensitive file from index.
  • git commit --amend: Fix last commit.
  • git rebase -i [commit_hash]: Rewrite history securely.
  • git-secrets: Prevent secret commits.
  • git-crypt: Encrypt sensitive files.
11. Git Rebase and Interactive Rebase

Git rebase allows you to modify commit history. Interactive rebase (git rebase -i) provides control to edit, squash, or reorder commits.

  • git rebase [branch]: Reapply commits onto new base branch.
  • git rebase -i [commit_hash]: Start interactive rebase session.
  • git rebase --continue: Continue after resolving conflicts.
  • git rebase --abort: Cancel rebase and restore branch.
  • git rebase --skip: Skip current commit during rebase.
  • git rebase --onto [new_base] [old_base]: Advanced rebase control.
12. Git Merge Conflict Resolution
  • git merge --no-ff [branch]: Merge without fast-forward.
  • git merge --abort: Abort current merge.
  • git mergetool: Launch merge conflict tool.
  • git diff --name-only --diff-filter=U: Show conflict files.
  • git status: Check merge conflict status.
  • git add [file]: Mark conflict as resolved.
  • git merge --squash [branch]: Merge and squash commits.
13. Git Commit Signing (GPG)

Commit signing ensures authenticity and integrity using cryptographic signatures.

  • git config --global user.signingkey [key_id]: Set GPG key.
  • git config --global commit.gpgsign true: Enable signing.
  • git commit --gpg-sign: Sign a commit.
  • git tag -s [tag_name]: Create signed tag.
  • git verify-commit [commit_hash]: Verify commit signature.
  • git verify-tag [tag_name]: Verify tag signature.
  • git log --show-signature: Show signature info.

  • git push --tags: Push release tags.

  • git merge --no-ff [branch]: Controlled release merge.
  • git branch -m [old] [new]: Rename branch.
